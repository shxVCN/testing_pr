## Сценарий 1: Создание опроса с вариантами ответов

**Название теста:** Проверка создания опроса и связанных вариантов в коллекции choices.

**Цель тестирования:** Убедиться, что авторизованный пользователь может создать опрос с несколькими вариантами ответов и что все варианты сохраняются в базе данных и связаны с опросом через коллекцию choices.

**Предпосылки:**
- Пользователь зарегистрирован и авторизован.
- Приложение и база данных работают нормально (миграции применены).
- Доступна страница создания опроса (/poll/create/).

**Шаги тестирования:**

1. Авторизоваться под тестовым пользователем (например, testuser).
   - **Ожидаемый результат:** Пользователь успешно авторизован.
2. Перейти на страницу создания опроса (/poll/create/).
   - **Ожидаемый результат:** Отображается форма создания опроса с полями: вопрос, вариант 1, вариант 2, вариант 3.
3. Заполнить форму: вопрос «Какой язык программирования лучше?», варианты: Python, JavaScript, Java. Отправить форму (POST).
   - **Ожидаемый результат:** Сервер возвращает редирект (код 302), опрос создаётся в БД с полем created_by = текущий пользователь.
4. Проверить в БД, что у созданного опроса в коллекции choices ровно 3 варианта с текстами Python, JavaScript, Java.
   - **Ожидаемый результат:** В коллекции poll.choices.all() содержится 3 объекта Choice с указанными текстами.

**Фактический результат:** Все шаги выполнены успешно. Тест test_poll_creation_with_choices (PollCreationTest) прошёл: редирект 302, опрос создан с правильным created_by, в коллекции choices ровно 3 варианта с ожидаемыми текстами.

## Сценарий 2: Голосование создаёт запись Vote и связывает с choice.votes

**Название теста:** Проверка того, что при голосовании создаётся запись Vote и она попадает в коллекцию votes выбранного варианта.

**Предпосылки:**
- Пользователь авторизован.
- В БД есть опрос с вариантами ответов (например, «Вариант A», «Вариант B»).
- Пользователь ещё не голосовал в этом опросе.

**Шаги тестирования:**

1. Авторизоваться под тестовым пользователем (voter).
   - **Ожидаемый результат:** Пользователь авторизован.
2. Перейти на страницу опроса (poll_detail по pk опроса).
   - **Ожидаемый результат:** Отображается форма голосования с вариантами.
3. Выбрать вариант «Вариант A» и отправить форму (POST с choice = id варианта A).
   - **Ожидаемый результат:** Сервер возвращает редирект (302).
4. Проверить в БД: у choice_a в коллекции votes одна запись, и user этой записи — текущий пользователь.
   - **Ожидаемый результат:** choice_a.votes.all() содержит 1 объект Vote с user = текущий пользователь.

**Фактический результат:** Тест test_vote_creates_vote_record (VoteLogicTest) прошёл: после POST получен редирект 302, в choice_a.votes одна запись с корректным user.

## Сценарий 3: Повторное голосование заменяет предыдущий голос

**Название теста:** Проверка правила «один голос на опрос»: при повторном голосовании старый голос удаляется, сохраняется только новый.

**Цель тестирования:** Убедиться, что пользователь может проголосовать только один раз в рамках одного опроса; при новом выборе предыдущий голос снимается с прежнего варианта и учитывается только новый.

**Предпосылки:**
- Пользователь авторизован.
- В БД есть опрос с вариантами A и B.
- У пользователя уже есть голос за вариант A (создан в БД).

**Шаги тестирования:**

1. Убедиться, что у пользователя есть голос за вариант A (Vote: choice=A, user=текущий пользователь).
   - **Ожидаемый результат:** choice_a.votes содержит голос данного пользователя.
2. От имени того же пользователя отправить POST на страницу опроса с выбором варианта B.
   - **Ожидаемый результат:** Редирект, голос за B учтён.
3. Проверить в БД: у варианта A нет голоса этого пользователя; у варианта B есть один голос этого пользователя.
   - **Ожидаемый результат:** choice_a.votes.filter(user=user).exists() == False; choice_b.votes.filter(user=user).exists() == True.

**Фактический результат:** Тест test_revote_replaces_previous_vote (VoteLogicTest) прошёл: после голосования за B у choice_a больше нет голоса пользователя, у choice_b голос пользователя присутствует.

## Сценарий 4: Страница результатов показывает корректные подсчёты голосов

**Название теста:** Проверка корректности подсчёта голосов на странице результатов и в модели (коллекции votes, get_total_votes, аннотация votes_count).

**Цель тестирования:** Убедиться, что количество голосов по каждому варианту и общее количество голосов по опросу отображаются и вычисляются правильно.

**Предпосылки:**
- В БД есть опрос с вариантами X и Y.
- За вариант X отдали голоса 2 пользователя, за вариант Y — 1 пользователь (итого 3 голоса).

**Шаги тестирования:**

1. Проверить в БД: choice_x.votes.count() == 2, choice_y.votes.count() == 1.
   - **Ожидаемый результат:** Подсчёты по вариантам верны.
2. Проверить метод опроса: poll.get_total_votes() == 3.
   - **Ожидаемый результат:** Общее количество голосов равно 3.
3. Проверить аннотацию как в представлении результатов: choices.annotate(votes_count=Count('votes')), сумма votes_count по всем вариантам равна 3.
   - **Ожидаемый результат:** Сумма votes_count по вариантам равна 3.

**Фактический результат:** Тест test_results_show_correct_counts (PollResultsTest) прошёл: подсчёты по choice_x (2) и choice_y (1), get_total_votes() = 3, сумма аннотированных votes_count = 3.

## Сценарий 5: Метод get_total_votes возвращает общее количество голосов

**Название теста:** Проверка метода модели Poll get_total_votes.

**Цель тестирования:** Убедиться, что метод get_total_votes() возвращает суммарное количество голосов по всем вариантам данного опроса.

**Предпосылки:**
- В БД есть опрос с хотя бы одним вариантом.
- По опросу есть одна или несколько записей Vote.

**Шаги тестирования:**

1. Создать опрос с одним вариантом и одним голосом. Вызвать poll.get_total_votes().
   - **Ожидаемый результат:** Возвращается 1.
2. Добавить ещё один голос за тот же вариант. Снова вызвать poll.get_total_votes().
   - **Ожидаемый результат:** Возвращается 2.

**Фактический результат:** Тест test_get_total_votes (PollModelMethodsTest) прошёл: при одном голосе метод вернул 1, после добавления второго — 2.

## Сценарий 6: Метод user_has_voted корректно определяет факт голосования

**Название теста:** Проверка метода модели Poll user_has_voted.

**Цель тестирования:** Убедиться, что метод user_has_voted(user) возвращает True, если пользователь уже голосовал в этом опросе, и False — если нет.

**Предпосылки:**
- В БД есть опрос с вариантами и хотя бы один голос от определённого пользователя.
- Есть другой пользователь, который не голосовал в этом опросе.

**Шаги тестирования:**

1. Для пользователя, который уже проголосовал в опросе, вызвать poll.user_has_voted(user).
   - **Ожидаемый результат:** True.
2. Для пользователя, который не голосовал в этом опросе, вызвать poll.user_has_voted(anonymous).
   - **Ожидаемый результат:** False.

**Фактический результат:** Тест test_user_has_voted (PollModelMethodsTest) прошёл: для проголосовавшего пользователя — True, для не голосовавшего — False.

## Сценарий 7: Коллекция choice.votes содержит все голоса за вариант

**Название теста:** Проверка коллекции votes у модели Choice (related_name и подсчёт).

**Цель тестирования:** Убедиться, что у варианта ответа (Choice) коллекция votes содержит все объекты Vote, отданные за этот вариант, и что подсчёт количества голосов (в т.ч. через get_votes_count при наличии) корректен.

**Предпосылки:**
- В БД есть опрос с одним вариантом (Choice).
- Несколько пользователей отдают голоса за этот вариант (например, 3 пользователя).

**Шаги тестирования:**

1. Создать 3 пользователей и для каждого создать Vote с choice = данный вариант.
   - **Ожидаемый результат:** В БД 3 записи Vote, связанные с этим choice.
2. Получить список голосов: choice.votes.all(). Проверить длину списка и при необходимости соответствие пользователей.
   - **Ожидаемый результат:** В списке 3 элемента, все с правильным choice.
3. Проверить метод choice.get_votes_count() (если есть) — должен вернуть 3.
   - **Ожидаемый результат:** get_votes_count() == 3.

**Фактический результат:** Тест test_choice_votes_collection (ChoiceVotesCollectionTest) прошёл: после добавления 3 голосов choice.votes.all() содержит 3 записи, get_votes_count() возвращает 3.
